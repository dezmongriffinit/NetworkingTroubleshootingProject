# Networking Troubleshooting Project

## Project Overview

This project demonstrates networking troubleshooting skills commonly used by IT Support and Help Desk professionals.

- TCP/IP
- DNS
- DHCP
- Ping Testing
- Network Connectivity Troubleshooting
- Router and Switch Concepts

---

## Scenario 1: No Internet Connection

### Problem

A user reports no internet access.

### Troubleshooting Steps

1. Verified physical connection.
2. Checked IP address configuration.
3. Tested gateway connectivity.
4. Performed ping tests.
5. Verified DNS resolution.

### Resolution

The network adapter had an incorrect IP configuration. Renewing the IP address restored connectivity.

---

## Scenario 2: DNS Failure

### Problem

Users can access websites by IP address but not by domain name.

### Troubleshooting Steps

1. Tested DNS resolution.
2. Checked DNS server configuration.
3. Flushed DNS cache.
4. Verified DNS server reachability.

### Resolution

An incorrect DNS server was configured. Updating DNS settings restored name resolution.

---

## Scenario 3: DHCP Issue

### Problem

A workstation receives an APIPA address (169.254.x.x).

### Troubleshooting Steps

1. Checked DHCP service availability.
2. Renewed IP address.
3. Restarted network adapter.
4. Tested network connectivity.

### Resolution

The DHCP service had stopped responding. Restarting the service restored proper IP assignment.
